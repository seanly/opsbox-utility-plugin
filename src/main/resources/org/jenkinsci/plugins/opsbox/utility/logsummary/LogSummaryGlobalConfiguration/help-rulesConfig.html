Log Summary Configuration

Configure log analysis rules in YAML format. The plugin will scan build logs and display matched lines in a summary table.

Configuration Format:

  • Rule Name: The key name for the rule (e.g., "error", "warning")
  • search: List of regex patterns to match log lines
  • showName: Display name shown in the summary table

Regex Pattern Tips:

  • (?i) - Case insensitive matching
  • ^ - Start of line
  • $ - End of line
  • .* - Any characters
  • \. - Literal dot (escaped)

Examples:

  • /(?i)^error / - Lines starting with "error " (case insensitive)
  • /[Ww]arning/ - Lines containing "Warning" or "warning"
  • /Exception/ - Lines containing "Exception"
  • /at .*\.java:/ - Stack trace lines with Java file references

Example Configuration:

error:
  search:
    - /(?i)^error /
    - /(?i)error:/
    - /(?i)fatal error/
    - /(?i)build failed/
    - /(?i)compilation failed/
  showName: Error

warning:
  search:
    - /[Ww]arning/
    - /WARNING/
    - /(?i)warning:/
    - /(?i)deprecated/
  showName: Warning

info:
  search:
    - /(?i)^info /
    - /(?i)information:/
    - /(?i)note:/
  showName: Information

exception:
  search:
    - /Exception/
    - /Error/
    - /Stack trace/
    - /at .*\.java:/
  showName: Exception
